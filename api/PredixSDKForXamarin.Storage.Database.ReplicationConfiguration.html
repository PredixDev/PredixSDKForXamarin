ReplicationConfiguration(Uri, Boolean, Boolean, ReplicationOptions, IDictionary<String, Object>, List<String>)

Initializes a ReplicationConfiguration structure with the provided parameters.

Defaults to a repeating, bi-directional configuration.

  • parameter url: URL to the Predix Sync offline database service instance
  • parameter isRepeating: This replication should be continous, or not.
  • parameter isBidirectional: Data replication should be both directions: client to server and server to client, or one-way: only server to client.
  • parameter options: An optional JSON-compatible dictionary of additional advanced properties that can be used to tweak replication.
  • parameter filterParameters: Optional parameters sent to the ReplicationFilterDelegate during document evaluation
  • parameter limitToChannels: A list of server-side defined channels.

IsRepeating

Indicates this replication should be continous, or not.

A repeating replication detects and automatically syncs data changes until explicitly stopped by calling stopReplication whereas a non-repeating replication syncs data with the service instance once then stops.

LimitToChannels

A list of server-side defined channels. If this list is empty, then all documents in all channels the user has access to are sent to the client.

If this property contains a list of channels, then only documents in those channels are sent to the client.

Example:**

User has access to channel1, channel2, and channel3

channel1 contains documents: docA, and docB

channel2 contains documents: docC, and docD

channel3 contains documents: docE, and docF

If LimitToChannels = {} then documents docA, docB, docC, docD, docE, and docF are sent to the client

If LimitToChannels = { &quot;channel1&quot;, &quot;channel3&quot; } then documents docA, docB, docE, and docF are sent to the client

If LimitToChannels = { &quot;channel4&quot; } then no documents are sent to the client (since the user doesn't have access to channel4
